Executive Summary

What we're building & why it wins

BADHIYA is a WhatsApp-native, voice-first, AI-powered business OS for the 63 million Indian MSMEs that every billing/ledger/payment app has ignored. The backend is already production-grade (NestJS + 7-layer AI pipeline + 12 migrations + 22 modules). This document plans the mobile app (Channel 2) — the app that the shopkeeper's next-gen inheritor "Priya" will live inside.

1

The Gap

90% of MSMEs accept digital payments. Only 12% are digitally mature. Every existing app owns 1-2 layers — billing (Vyapar), ledger (Khatabook), payments (PhonePe). None stacks AI advisor + WhatsApp-native + voice + health score + NBFC marketplace + ONDC + gamification.

2

The Whitespace

Research confirmed no incumbent holds more than 2 of these axes. Pilloo AI (voice-AP only), SayBill (WA+voice-narrow), Meta-MoMSME chatbot (generic, not transactional). Badhiya has a ~12 month window before these mature a second axis.

3

The Bet

Ship on Expo SDK 55 + New Architecture + Hermes V1 + Legend State v3 + Drizzle + Victory XL + Maestro. Make it feel like CRED meets WhatsApp meets Khatabook meets a bahi-khata page. Voice is the hero, not the keyboard.

The one-sentence positioning: Badhiya is the only MSME mobile app where Rajesh the kirana owner can say "Sharma ji ko 5 kilo aata 250 rupaye ka bill bana do" in Hindi, and the app records the sale, generates a GST invoice, updates inventory, sends the invoice on WhatsApp, updates the health score, recommends a PMEGP loan if eligible — all in under 4 seconds, offline-capable, and gamified.
Baseline Analysis

CourtNetra — what we must beat

CourtNetra is a production legal-practice-management mobile app (21 releases, v1.0.0 → v2.1.0). It's a solid reference for the class of app we're building. Here's what they use, and where Badhiya will be strictly better.

CourtNetra's Stack (Nov 2025 snapshot)

  • Expo SDK 54 · expo-router 6 · React Native 0.81 · React 19.1
  • Zustand for client state (auth, timer, features, language, client-auth)
  • TanStack Query v5 for server data
  • Axios for HTTP
  • Reanimated 4 + react-native-worklets
  • react-native-gifted-charts for charts (SVG-based)
  • expo-sqlite for offline
  • expo-av for audio (being removed in SDK 55)
  • expo-local-authentication, expo-notifications, expo-haptics
  • File structure: app/ (routes) + src/ (components, hooks, lib, stores, theme)

Where Badhiya Leapfrogs

  • Expo SDK 55 (RN 0.83, React 19.2) — one generation ahead, Hermes V1 opt-in, OTA diffing
  • Legend State v3 (signals + built-in sync) replaces Zustand — no DIY mutation queue
  • Drizzle ORM on expo-sqlite — type-safe queries, declarative migrations, useLiveQuery reactivity
  • Victory Native XL (Skia-backed, GPU) replaces gifted-charts — 60fps on ₹8K phones
  • expo-audio replaces removed expo-av
  • expo-background-task replaces deprecated expo-background-fetch
  • expo-speech-recognition + Bhashini server hybrid for Hindi STT
  • react-native-vision-camera + ML Kit OCR for receipt scanning
  • react-native-mmkv for 30x faster KV storage
  • Maestro for E2E (CourtNetra has none visible)
  • New Architecture on from day 1 — no migration debt
  • Full i18n with i18next — Hindi default, 11-language ready
Hard migration constraint: expo-av is removed in SDK 55. CourtNetra will have to migrate audio to expo-audio soon. Badhiya starts fresh on SDK 55, so we skip that tax entirely.

Version Pinning Strategy

🔒 Strict pinning (exact)

  • expo — exact version, let npx expo install resolve everything managed
  • react-native — managed by Expo resolver
  • All expo-* packages — npx expo install <pkg>, never freelance

~ Tilde (patch only)

  • Third-party native libs: Reanimated, Gesture Handler, Skia, Vision Camera, Drizzle, Legend State, MMKV
  • Major bumps require a planning ticket, never opportunistic

^ Caret (minor allowed)

  • Pure JS libs: zod, react-hook-form, i18next, axios, TanStack Query
  • Stack bumps: one cadence per quarter, scheduled, never ad-hoc

🚢 OTA channels

  • production — pinned to latest native binary-compatible JS bundle
  • staging — diffing tests, internal QA
  • dev — canary builds, feature flags
Migration risks (blocking):
  1. expo-av is GONE in SDK 55 — must use expo-audio from day 1 (we are).
  2. Old Architecture is gone — every native dep must be New-Arch-compatible. Audit before installing.
  3. Xcode 26 required for SDK 55 iOS builds. CI and contributors must update.
  4. Hermes V1 opt-in on SDK 55 — test thoroughly before flipping; SDK 56 makes it default.
  5. Drizzle migrations on device — write a Maestro test for "open v1 data with v2 binary" path.
  6. CocoaPods EOL Dec 2 2026 — plan SPM migration by end-2026.
Design System

Warm, trustworthy, distinctly Indian

Built for Rajesh (38-52 year old kirana owner, 10th pass, Hindi-first, low digital literacy, ₹10K Android). Design philosophy: quiet trust over loud marketing. Saffron is an accent, never a flood. Every status has color + icon + Hindi label. Voice is the hero, keyboard is the escape hatch.

15 UI/UX Principles (non-negotiable)

1

Digitize existing habits

Every screen maps 1:1 to paper-world: ledger page, cash drawer, daily diary. No new metaphors invented.

2

Voice > vernacular text

Hindi translations fail ("anivarya" lost to "zaroori"). Voice-first, copy written by Tier-3 speakers, never translated.

3

Icon + text label always

Bare icons fail with low-literacy users. Every tap target gets a Hindi label. No exceptions.

4

56dp minimum touch targets

Material says 48dp, Apple 44pt. For greasy thumbs in poor light during customer rush, we go 56-64dp minimum for primary actions.

5

WhatsApp is the UX vocabulary

Double-ticks = synced. Voice-note waveform = voice input. Status circles. Forward arrows. Reuse what they already know.

6

Photo + OCR + voice > typing

Typing Hindi on a ₹10K phone is brutal. Default to camera, voice, and barcode. Typing is the last resort.

7

Quiet trust over loud marketing

CRED proved calm = trustworthy for money apps. Saffron as accent, not flood. Deep whitespace, restrained motion.

8

Color + icon + label for status

Red/green colorblindness is common. Always pair color with arrow (▲▼), label (Munafa/Nuksaan), and absolute number.

9

One primary action per screen

Decision overload is fatal at 10th-pass literacy. One big saffron CTA at bottom, one optional secondary action.

10

Bottom sheets, never modal hell

Sheets feel like drawers. Preserve context behind them. ChatGPT, GPay, PhonePe all use them as default disclosure.

11

Offline-first is non-negotiable

2G/3G dead zones, broken WiFi, exhausted prepaid are normal. 100% functional offline, sync silently in background.

12

AI as invisible layer, not chat

AI responses render as generative UI cards — a chart, an action button, a profit summary. Never a wall of text.

13

Multi-sense confirmation

Every money action = visual check + haptic success + Hindi voice "ho gaya, badhiya!" + sound chime.

14

Zerodha-grade number density

Monospaced numerals, right-aligned rupees, color-coded, generous whitespace around digits. Never let numbers collide.

15

Onboarding via dukan, not forms

Paytm loses users at OTP/KYC. Badhiya onboards with a single voice prompt → photo → done. KYC happens lazily.

Animation Philosophy

Warm, never flashy. Saffron is already attention-grabbing; motion should feel like turning a ledger page, not a video game.

  • gentle (damping 20, stiffness 90) — screen entry, sheets
  • snappy (damping 15, stiffness 200) — button feedback, tabs
  • bounce (damping 8, stiffness 180) — ONLY for success, never errors

6 core animations: number tickers (Skia odometer), pulse orb (AI listening), voice waveform, page-turn (khata detail), checkmark reveal, saffron-tinted shimmer (never gray).

Offline-First Architecture

Core principle: Local DB is the source of truth. Server is a sync target, not the master.

Data Flow

  1. User action (voice → STT → form) writes to Drizzle + expo-sqlite immediately
  2. Record marked _status: "created", UI updates via Legend State observable
  3. Haptic + sound + checkmark fires instantly (no network wait)
  4. expo-background-task runs sync every 15 min on WiFi/cellular
  5. Pulls server changes since lastSyncedAt, pushes local dirty records
  6. Conflicts resolved per field-level rules (never naive LWW)

Conflict Resolution

  • Khata txns: append-only log → CRDT-friendly, no conflicts
  • Customer details: field-level merge (name local + phone server both survive)
  • Inventory counts: server authoritative + "confirm count?" UI
  • Settings: device-local, never synced

Storage Budget (₹8K Android)

  • Drizzle + SQLite: ~200KB per 1k txns → 5 years kirana = ~50MB
  • MMKV cache: ~5MB
  • Voice notes: 50MB cap, auto-purge after sync
  • Photos: thumbnails local, originals server-side lazy load

Roadmap

10 phases · ship in logical order

Backend is production-ready. Frontend web dashboard is built. This plan ships the mobile app (Channel 2) on a phased roadmap — de-risk the unknowns first (voice, offline sync, native builds) before feature-building. Each phase is a closable milestone with its own Maestro test suite and internal demo.

Phase 0 · Complete

Baseline research & planning ✓

Three research agents dispatched (competitors, UX, advanced stack). 5,694 lines of existing docs reviewed. CourtNetra stack analyzed. This HTML plan committed.

Phase 1 · Sprint 1 (current)

Scaffold + risk de-risking

Bootstrap Expo SDK 55 app. Wire the riskiest unknowns first so surprises happen early when they're cheap.

  • npx create-expo-app with New Arch + Hermes V1 opt-in, SDK 55 pinned
  • Drizzle + expo-sqlite + Legend State signed off on a throwaway "hello khata" flow
  • expo-audio + Bhashini hybrid STT proof — record a Hindi sentence, get text back, end-to-end
  • EAS Build staging channel, first OTA push tested
  • Maestro installed, first flow "app opens without crashing on ₹8K Redmi"
  • Sentry + PostHog wired with PII masking
  • Cold boot on real Redmi A-series <2.5s or fix before any feature work
Phase 2 · Sprint 2

Design system + navigation shell

Theme tokens, typography, the 5-tab shell, bottom sheets, and one fully-polished Home screen as the pattern everyone else follows.

  • Colors, Spacing, Radius, FontSize tokens (from design system above)
  • Hind + Anek Devanagari + JetBrains Mono font loading
  • expo-router 6 layout — (auth) and (tabs) groups
  • HapticsService centralized — 8 patterns
  • 6 core animations (Reanimated 4): ticker, orb, waveform, page-turn, checkmark, shimmer
  • Reusable primitives: KpiTile, TxnRow, BottomSheet, PrimaryCTA, ChipRow, LoadingSkeleton
Phase 3 · Sprint 3

Auth + onboarding flow

Voice-first onboarding. Connect to existing backend POST /auth/otp/send and /auth/otp/verify. Biometric lock.

  • Splash + brand reveal
  • Phone + OTP screen (10-digit validation, +91 prefix, auto-OTP from SMS)
  • Voice onboarding wizard — 3 steps (shop name, category, location)
  • JWT + refresh token stored in expo-secure-store
  • Legend State auth store persisted in MMKV
  • Biometric gate on app open after 10 min idle
  • Maestro suite: full onboarding happy path
Phase 4 · Sprint 4

Home + Transactions

Morning briefing dashboard. Voice-record a sale end-to-end with optimistic UI. Wire offline-first sync.

  • Home screen: KPI bento + txn list + AI pill
  • Voice input sheet (global, summonable)
  • Sale/expense bottom sheet with Zod-validated form
  • Backend POST /transactions + GET /pnl/today wired via TanStack Query
  • Drizzle schema + migrations for transactions table
  • Legend State syncedFetch for offline mutation queue
  • Maestro: "record a sale offline, kill app, reopen, verify synced"
Phase 5 · Sprint 5

Khata (credit ledger)

The killer feature for kirana shopkeepers. Digitized bahi-khata with swipe-to-remind via WhatsApp deep links.

  • Customer list with color-coded aging
  • Customer detail ledger (paper metaphor)
  • Give credit / record payment sheets
  • WhatsApp reminder deep link via wa.me with pre-filled Hindi template
  • Bulk reminder via backend POST /credit/remind/all
  • Overdue calculation on-device (fast)
  • Maestro: "create customer, give credit, send reminder"
Phase 6 · Sprint 6

Inventory (Dukan)

Photo-first product grid. Vision Camera + ML Kit OCR for receipt scanning. Low-stock alerts.

  • Product grid with filter chips
  • Add product via camera → OCR → voice confirm price
  • Stock update bottom sheet
  • Low-stock alert integration with backend BullMQ
  • Barcode scanning via Vision Camera frame processors
  • Maestro: "scan a product box, extract name + price, save"
Phase 7 · Sprint 7

AI Chat · Pucho Kuch Bhi

Generative UI responses connected to the 7-layer backend pipeline. Voice in, voice out, rendered as cards.

  • Chat UI with pulsing orb animation (Skia)
  • Generative response renderer — routes to card components based on intent
  • Backend POST /ai/query wired (already built)
  • TTS via expo-speech with Hindi voice
  • Suggestion chips that auto-submit
  • Offline: queues queries, fires when connectivity returns
  • Maestro: "ask P&L, verify card renders, tap apply loan"
Phase 8 · Sprint 8

Paisa · Health Score · Finance Hub

Victory Native XL charts. Skia gauge. Loan marketplace + scheme matcher from backend.

  • Revenue area chart with gesture scrubbing
  • Skia-rendered 0-900 health score gauge
  • 6-component breakdown with animated bars
  • NBFC loan marketplace cards (5 lenders)
  • Govt scheme cards with 1-tap apply flow
  • Backend /health-score/breakdown, /schemes, /loans wired
Phase 9 · Sprint 9

ONDC + GST Invoicing

Seller dashboard for ONDC. GST invoice generation with GSTR-1/3B auto-filing. WhatsApp share.

  • ONDC seller onboarding (backend already has the module)
  • Order acceptance flow with notification
  • GST invoice template rendering
  • PDF export via expo-print
  • WhatsApp deep link share with PDF attached
  • GSTR filing history screen
Phase 10 · Sprint 10

Gamification + Notifications + Polish

Streaks, badges, leaderboards. Daily push notifications via expo-background-task + FCM v1. Final polish.

  • Rewards screen with streak tracker + badges + city leaderboard
  • Notifications screen with 3 BullMQ-driven categories
  • expo-background-task for daily sync + briefing scheduling
  • Lottie success animations for milestones
  • i18next with en/hi message files (from existing frontend)
  • Play Store ASO: screenshots, description, Hindi metadata
  • Internal beta via EAS Update dev channel
  • Full Maestro regression suite
Phase 11 · Launch

Closed beta → Play Store

Ship to 100 shopkeepers in Noida first. Measure streak adherence, voice accuracy, crash-free rate. Iterate before public launch.

  • 100 seed users via direct outreach + WhatsApp
  • Sentry crash-free target: 99.5%+
  • PostHog funnels: onboard → first sale → first WhatsApp reminder → first AI query
  • Weekly cohort retention target: 40%+ W1, 25%+ W4
  • Play Store staged rollout: 5% → 25% → 100% over 3 weeks
  • iOS build via TestFlight (optional — Android is primary)
